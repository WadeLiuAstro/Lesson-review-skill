# 转写稿精修为深度复习笔记

将 Whisper 自动转写的课堂口语文本，精修为结构化、可直接用于期末复习的深度笔记。本技能提供内容深度的标准和方法论，`course-review-pipeline` 负责流水线编排（扫描、合并、材料提取、索引），两者互补使用。

## Codex 兼容说明

- 本技能为 Codex 兼容副本，目标是在不影响 Qoder 原版的前提下做最小兼容改造。
- 法条与案例查询优先使用当前环境可用的法律数据库或 MCP（如 `pkulaw`）；若当前会话没有专用法律工具，只能引用已被课程材料、现有笔记或可靠上下文明确确认的内容，不能猜条号、补原文。
- 如需保留待核信息，统一标记为 `需回看确认`，避免把未经核实的法条或案例写成确定结论。
- Step 7 中的并行代理仅在**用户明确要求**且环境支持时启用；否则由当前代理顺序执行相同标准的精修流程。

## 核心原则

1. **从口语到书面，保留课堂洞察**：转写稿是口语化的，但教师在课堂上强调的重点、举的例子、对比的分析往往是最有价值的考点。精修不是删除口语，而是提炼这些洞察并用书面语重新组织。
2. **概念优先，法条佐证**：先讲清楚制度的"为什么"（制度逻辑），再引用法条作为权威支撑。不要堆砌法条而忽略解释。
3. **以考试为导向**：每份笔记都应让读者能直接用于答题——看到题目后能快速定位知识点、回忆法条、运用分析框架。

## Step 1: 转写稿分析与知识结构化

### 从口语中提取知识点骨架

读取转写稿后，按以下维度标记内容：

| 维度 | 识别线索 | 处理方式 |
|------|---------|---------|
| 概念定义 | "所谓……就是""它的本质是" | 提炼为精确定义，补充构成要件 |
| 制度逻辑 | "为什么要这样规定""之所以……是因为" | 展开为制度设计的深层原因分析 |
| 对比分析 | "和……不同""区别在于""对比一下" | 构建对比表格或分点阐述 |
| 争议焦点 | "有争议""两种观点""学界认为" | 整理为学说争议，标注主流/少数观点 |
| 教师强调 | "这个很重要""考试可能会考""注意" | 高优先级纳入 exam_focus |
| 案例讨论 | 具体案件名称、事实描述 | 提炼为案例分析框架（见 Step 5） |

### 输出结构

精修后的笔记应包含清晰的层级：

```
# [主题名]

## 一、核心概念与制度逻辑
  （概念定义 + 制度设计的"为什么"）

## 二、构成要件/法律框架
  （要件分解 + 法条引用）

## 三、关键制度/规则详解
  （具体规则的深入分析）

## 四、学说争议与比较
  （如有争议，列出不同观点）

## 五、案例分析
  （课堂讨论的案例 + 分析框架）

## 六、考试要点
  （高频考点 + 答题思路）
```

## Step 2: course_summary 与 exam_focus 的写作规范

### course_summary — 知识全景图

定位为**完整的学习笔记**，覆盖课堂所有知识点，适合系统复习。内容侧重：
- 概念定义的精确表述
- 制度逻辑的展开分析
- 法条体系的完整引用
- 学说对比与争议梳理
- 图片的概念拓展式说明（见 Step 3）

### exam_focus — 考试速查手册

定位为**考前突击的速查工具**，只保留高频考点和答题框架。内容侧重：
- 核心考点清单（用星级标记重要程度）
- 法条速查表（条号 + 一句话要点）
- 答题模板（见 Step 6）
- 易混淆概念的快速区分表
- 常见陷阱提示

### 两种文件的关系

`course_summary` 是"学得懂"，`exam_focus` 是"答得好"。前者详尽，后者精炼。exam_focus 中的每个要点都应能在 course_summary 中找到对应的详细展开。

## Step 3: 概念拓展式图片说明

这是精修的核心方法论。幻灯片图片不应只是"贴图"，而应成为讲解概念的载体。

### 核心写法

**三步结构**：

1. **先抛出"为什么"的问题** — 引发思考，建立问题导向
2. **借图片讲清制度逻辑** — 用图片作为讲解的锚点，展开分析
3. **延伸到实务或考试要点** — 连接到具体的法律适用或答题要求

### 差的写法（描述模式）

> PPT 展示了信用证的 12 步流程图。
>
> PPT 用三角形结构图展示了融资租赁的三方当事人。
>
> PPT 明确了内保外贷的四方主体。

这种写法的问题是：只是告诉读者"图里有什么"，没有帮助读者理解"为什么是这样的"。

### 好的写法（概念拓展模式）

**示例 1：信用证流程**

> 为什么信用证需要 12 步之多？核心在于买卖双方互不信任——买方怕付了钱拿不到货，卖方怕发了货收不到钱。银行作为信用中介介入后，形成了三条分离的线路：合同线（买卖双方）、货物线（运输和保险）、单据线（银行间的单据传递）。下图展示了这一机制如何运作：
>
> ![信用证 12 步流程](slides/topic02-slide-15.png)
>
> 其中最关键的环节是"议付"和"押汇"——议付行审核单据后先行垫付资金给受益人，承担了开证行拒付的风险。UCP 600 第 14 条规定了 5 个工作日的审单时限，这个时限的设计就是为了平衡"充分审查"和"及时付款"之间的张力。

**示例 2：融资租赁三方关系**

> 融资租赁和普通租赁的本质差异在哪？普通租赁是"我有设备，租给你用"；融资租赁是"你要什么设备，我出钱买，再租给你"。这导致三方关系——出租人、承租人、出卖人——形成了一个特殊的交易结构：
>
> ![融资租赁三方关系](slides/topic06-slide-08.png)
>
> 出租人虽然是"买设备的人"，但设备是承租人自己选的，所以出租人不承担物的瑕疵担保责任（《民法典》第 747 条）。租金也不是"使用费"，而是"融资对价"——本质上是出租人替承租人垫付设备款后，分期收回本息。

**示例 3：内保外贷**

> 境内母公司想给境外子公司融资，但不能直接把钱汇出去（外汇管制）。怎么办？通过"内保外贷"——母公司在境内银行开保函，境外银行凭保函给子公司贷款。信用传导路径是：境内母公司信用 → 境内银行保函 → 境外银行贷款 → 境外子公司。
>
> ![内保外贷四方主体与信用传导](slides/topic02-slide-22.png)
>
> 这里的关键法律问题是"见索即付"原则：境外银行只要提交了符合保函要求的索赔文件，境内银行就必须付款，不需要审查基础合同的履行情况。这大大降低了境外银行的信贷风险，但也使境内银行面临"恶意索赔"的风险。

### 不同法学领域的写法变体

- **信托法**：侧重"法律关系分析"——委托人、受托人、受益人的权利义务关系，财产独立性的法律效果
- **竞争法/反垄断法**：侧重"行为认定"——构成要件分析、相关市场界定、竞争效果评估
- **国际金融法**：侧重"交易结构"——多方当事人的信用链条、风险分配机制
- **公司法**：侧重"治理结构"——股东会/董事会/监事会的权力配置、信义义务

## Step 4: 法条查询与嵌入

### 法律检索工具使用

优先使用当前环境可用的法律数据库、MCP 或检索工具查询法律条文与案例；`pkulaw` 只是推荐选项之一，不应视为当前会话中的硬依赖。

- **有专用法律工具时**：优先用检索工具核对条号、法规名称、案例名称，再决定是否引用原文。
- **无专用法律工具时**：只引用已被 transcript、现有讲义、教师口述或项目内可靠材料明确支持的内容；不补写不确定的法条原文，不猜测案例案号。
- **存在不确定性时**：保守处理，在笔记中标记 `需回看确认`，把核验任务留给后续人工或具备法律数据库的会话。

### 查询流程

```
1. 从转写稿中识别教师提到的法律名称、条号和案例线索
2. 若有专用法律工具，先核对条号与法规名称是否正确（教师口误或转写错误常见）
3. 仅在内容已核实的前提下嵌入法条原文或摘要
4. 如果转写稿提到案例，优先用可用法律工具检索；若无法核实，仅保留课堂层面的事实和启示，不补写未经确认的裁判细节
```

### 引用格式

在笔记中嵌入法条时，使用以下格式：

```markdown
> **《信托法》第 2 条**：本法所称信托，是指委托人基于对受托人的信任，
> 将其财产权委托给受托人，由受托人按委托人的意愿以自己的名义，
> 为受益人的利益或者特定目的，进行管理或者处分的行为。

要点提炼：信托的四个核心要素——信任基础、财产委托、受托人名义、受益人利益。
```

注意：引用法条后必须有**要点提炼或解释**，不能只贴原文不解释。

### 法条嵌入位置

法条应嵌入在讲解相关知识点的段落中，而非集中罗列。好的做法是在讲完制度逻辑后紧跟法条引用作为权威支撑，形成"制度逻辑 → 法条依据 → 实务适用"的三段结构。

## Step 5: 案例分析框架构建

### 四段结构

每个案例按以下结构组织：

```markdown
### [案例名称]

**事实**：（2-3 句话概括关键事实，不需要完整复述案情）

**争点**：（本案的核心法律问题是什么？通常 1-2 个）

**裁判**：（法院怎么判的？依据什么法条？关键推理是什么？）

**启示**：（对制度理解或考试答题有什么帮助？与课堂讨论的联系）
```

### 从转写稿中提取案例

教师在课堂上讨论案例时，通常会包含以下信息：
- 案件名称或当事人（可能在口语中不完整）
- 关键事实的口头描述
- 教师对判决的评价（"这个判决对/不对""合理的做法是"）

提取时注意：
1. 如果案件名称不完整，优先用当前可用的法律检索工具根据关键事实搜索；若暂时无法核实，保留课堂表述并标记 `需回看确认`
2. 教师的评价往往是最有价值的"启示"部分，要重点提炼
3. 如果教师将案例与特定法条关联，保留这个关联

## Step 6: 答题模板生成

### 常见题型与对应模板

**概念辨析题**：
```
1. 分别定义 A 和 B
2. 指出核心区别（通常 2-3 个维度）
3. 举例说明
4. 总结：为什么区分重要（法律后果不同）
```

**构成要件分析题**：
```
1. 列出所有构成要件
2. 逐一分析每个要件的认定标准
3. 引用对应法条
4. 结合事实判断是否满足
```

**案例分析题**：
```
1. 识别法律关系（当事人之间是什么关系？）
2. 定位争议焦点（争什么？）
3. 适用法条（用什么规则判断？）
4. 得出结论（结论 + 理由）
```

**制度比较题**：
```
1. 概述两种制度的基本定位
2. 从 3-4 个维度对比（主体、客体、效力、程序等）
3. 分析差异的制度原因
4. 评价优劣或适用场景
```

### 模板在笔记中的位置

答题模板放在 `exam_focus` 的对应章节末尾，或在 course_summary 的"考试要点"部分。每个模板应与具体的知识点关联，而非独立成节。

## Step 7: 分批处理

### 板块划分策略

按课程内容将 sessions 分为 2-4 个板块。若用户明确要求并行代理且环境支持，可一板块一代理并行精修；否则由当前代理按板块顺序完成：

| 板块 | 内容特征 | 典型 sessions 数 |
|------|---------|-----------------|
| A：总论/基础 | 概述、设立要件、基本原则 | 3-5 |
| B：法律关系深化 | 权利义务、财产独立性、核心制度 | 3-5 |
| C：专题一 | 具体制度、监管规则 | 3-5 |
| D：专题二 | 实务案例、前沿问题 | 3-5 |

板块数量取决于课程总 session 数。一般 10 个 session 以下分 2 批，10-20 个分 3 批，20 个以上分 4 批。

### 并行代理启用时的 prompt 构建要点

每个并行代理的 prompt 必须包含：

1. **明确的任务边界**：负责哪些 session（编号范围），不多不少
2. **文件路径清单**：转写稿路径、现有笔记路径、幻灯片图片路径、复习资料文本路径
3. **精修标准摘要**：核心原则 + 概念拓展式图片写法 + 法条引用格式（从本技能提取关键段落，不需要复制全文）
4. **输出要求**：明确说明要产出 course_summary 和 exam_focus 两份文件
5. **工作流指示**：先读现有笔记 → 再读转写稿 → 用 Edit 工具逐份补充 → 完成后报告修改摘要

### 并行代理 prompt 模板

详见 [reference.md](reference.md) 中的"并行代理 prompt 增强模板"。

## 质量检查清单

每份精修后的笔记应满足：

```
- [ ] course_summary 覆盖了转写稿中的所有重要知识点
- [ ] exam_focus 包含高频考点清单和答题模板
- [ ] 图片说明采用概念拓展模式（非描述模式）
- [ ] 每个图片引用都准确对应描述内容（用 Read 打开图片验证）
- [ ] 法条引用准确（优先通过可用法律数据库验证；无专用工具时仅引用已可靠确认内容）
- [ ] 法条引用后附有点评或要点提炼
- [ ] 案例分析采用四段结构（事实→争点→裁判→启示）
- [ ] 答题模板与具体知识点关联
- [ ] 两种文件之间要点可互相映射
```

## 与 course-review-pipeline 的配合

`course-review-pipeline` 负责流水线编排（Step 1-6），本技能在 pipeline 的 Step 5（分批补充笔记）时提供精修深度标准。

### 配合方式

1. **pipeline 先执行 Step 1-4**：扫描目录、合并 session、提取材料、识别关键幻灯片
2. **加载本技能**：在 Step 5 的执行说明中引用本技能的核心原则和写法标准；如启用并行代理，则写入对应 prompt
3. **pipeline 继续 Step 6**：质量检查（图片引用验证）和索引生成

### 独立使用

本技能也可独立使用，适用于以下场景：
- 已有转写稿但没有 PPT/PDF 复习资料（跳过图片嵌入，专注文本精修）
- 只需精修部分 session 而非全部课程
- 非课程类的转写稿精修（如会议记录、讲座转录）

独立使用时，仍需遵循本技能的核心原则和写法标准，但不需要 pipeline 的目录结构约定。
