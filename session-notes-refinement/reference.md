# reference.md — session-notes-refinement 辅助文件

本文档包含概念拓展式写法的详细示例集、法条查询模板（`pkulaw` 优先但可降级）、并行代理 prompt 增强模板，供精修时按需引用。

## 1. 概念拓展式写法示例集

### 1.1 信托法领域

**差的写法**：
> PPT 展示了信托法律关系的三角形结构图。

**好的写法**：
> 信托法律关系为什么被称为"最精巧的财产管理制度"？关键在于它创造了一个独特的三方结构——委托人设立信托后"退出"了法律关系，受托人以自己的名义管理财产，而受益人享有利益但不承担管理责任。这种"所有权与利益分离"的设计，在大陆法系的"一物一权"传统下显得尤为特殊：
>
> ![信托三方结构](slides/trust-slide-05.png)
>
> 正是这种分离，引出了信托法最核心的问题：受托人如果违背信托目的处分财产，受益人能否追回？这涉及信托财产的"追及效力"问题（《信托法》第 22 条），也是信托区别于一般委托关系的关键。

---

**差的写法**：
> 老师讲了信托财产独立性的概念。

**好的写法**：
> 信托财产独立性是整个信托制度的基石——它意味着信托财产既不属于委托人的遗产，也不属于受托人的固有财产，更不是受益人的责任财产。这种"三重隔离"在法律上如何实现？
>
> 核心法条是《信托法》第 15-18 条构成的独立性体系：第 15 条隔离委托人的债权人（委托人破产时信托财产不纳入清算），第 16 条隔离受托人的固有财产（受托人死亡或破产时信托财产不受影响），第 17 条规定信托财产强制执行的四项例外情形。
>
> 考试最常考的是第 17 条的四项例外：（1）设立信托前已存在的优先债权；（2）信托事务处理中产生的债务；（3）信托财产本身应负担的税款；（4）法律规定的其他情形。注意区分：前两项是"信托财产自身产生的债务"，后两项是"法定例外"。

### 1.2 竞争法/反垄断法领域

**差的写法**：
> 老师讲了横向垄断协议的构成要件。

**好的写法**：
> 横向垄断协议为什么被称为"核心卡特尔"？因为它是同一产销阶段的竞争者之间达成的协议，直接消灭了竞争。与纵向协议（上下游企业之间）不同，横向协议的反竞争效果几乎是"当然"的，所以各国反垄断法对其采取最严厉的规制态度：
>
> ![横向垄断协议类型](slides/antimonopoly-slide-03.png)
>
> 《反垄断法》第 17 条列举了五类横向协议：固定价格、限制产量、分割市场、限制新技术、联合抵制交易。注意一个关键区别：前四类是"行为本身违法"（per se illegal），第五类联合抵制则需要证明"排除、限制竞争的效果"。这个区分在考试中经常出现。

### 1.3 国际金融法领域

**差的写法**：
> PPT 展示了项目融资的风险分配表。

**好的写法**：
> 项目融资和普通贷款的根本区别是什么？普通贷款看的是借款人的整体信用（"你有多少资产"），项目融资看的是项目本身的未来收益（"这个项目能赚多少钱"）。这个区别导致了完全不同的风险分配逻辑：
>
> ![项目融资风险分配表](slides/topic02-slide-30.png)
>
> 上图中的纸浆厂案例很好地说明了这一点：项目建设阶段的风险由承包商承担（完工担保），运营阶段的原料供应风险由供应商承担（长期供应合同），市场销售风险由购买方承担（照付不议合同）。贷款银行的策略是"把所有风险分配给最能控制该风险的一方"——这就是项目融资的核心设计哲学。

### 1.4 公司法领域

**差的写法**：
> 老师讲了董事的信义义务。

**好的写法**：
> 董事的信义义务（fiduciary duty）为什么分为"忠实义务"和"勤勉义务"两条线？因为两者防范的风险不同：忠实义务防范的是"利益冲突"（董事拿公司和自己的利益做交换），勤勉义务防范的是"消极怠工"（董事不认真履职）。
>
> 《公司法》第 180 条规定了忠实义务的核心规则：董事不得利用职权谋取属于公司的商业机会、不得自营或为他人经营与公司同类的业务、不得违反章程规定与本公司进行交易。这三条本质上都是解决"利益冲突"问题——确保董事在做决策时，考虑的是公司利益而非个人私利。

## 2. 法条查询模板（pkulaw 优先，可降级）

### 2.1 精确查询已知法条

当转写稿中教师提到了具体法条号时：

```
工具：get_article
参数：
  - law_name: "信托法" 或 "中华人民共和国信托法"
  - article_number: "17"
  
结果：获取第 17 条完整原文
```

### 2.2 关键词搜索法条

当不确定具体条号时：

```
工具：search_article
参数：
  - keyword: "信托财产 强制执行"
  - law_name: "信托法"（可选，缩小范围）
  
结果：返回匹配的法条列表，找到正确条号后再用 get_article 获取全文
```

### 2.3 案例搜索

当转写稿提到案例但信息不完整时：

```
工具：search_case
参数：
  - keyword: "受益权 强制执行 信托"
  
结果：返回相关案例列表
```

### 2.4 常见法条领域速查

| 课程领域 | 常用法律 | 核心条号范围 |
|---------|---------|------------|
| 信托法 | 《信托法》 | 第 2 条（定义）、第 6-13 条（设立）、第 14-18 条（财产独立性）、第 22-26 条（受托人义务）、第 43-49 条（受益人权利） |
| 竞争法 | 《反不正当竞争法》 | 第 2 条（一般条款）、第 6-12 条（具体不正当竞争行为） |
| 反垄断法 | 《反垄断法》 | 第 16-21 条（垄断协议）、第 22-24 条（滥用市场支配地位）、第 25-38 条（经营者集中） |
| 国际金融法 | UCP 600、《票据法》 | UCP 600 第 2/14/15 条（信用证审单）、《票据法》第 10-13 条（票据行为） |
| 公司法 | 《公司法》（2023 修订） | 第 180-184 条（信义义务）、第 89-91 条（股东知情权）、第 191-193 条（公司决议效力） |

注意：条号可能因法律修订而变化。若当前会话可用 `pkulaw` 或其他法律数据库，应优先核对最新版本；若无专用工具，只能引用已被当前材料可靠确认的内容，不确定时标记 `需回看确认`。

## 3. 并行代理 prompt 增强模板

### 3.1 完整精修代理 prompt

以下模板仅在 `course-review-pipeline` Step 5 明确启用并行代理时使用；若当前会话按顺序执行，可直接复用其中的精修标准，不必强制拆成多个代理。

```
你的任务是对 {课程名} session_notes 第 {起始}-{结束} 讲进行精细化深度改写。

## 项目导向
期末复习辅助项目。精修后的笔记应能直接用于考试答题。

## 你负责的文件

| Session | 日期 | 教师 | 主题 |
|---------|------|------|------|
{session_table}

## 文件路径
- 当前 session_notes：`{notes_path}`
- 转写稿：`{sessions_path}`
- 幻灯片图片：`{slides_path}`（如有）
- 复习资料文本：`{extracted_text_path}`（如有）

## 精修标准

### 核心原则
1. 从口语中提炼知识点，用书面语重新组织，保留课堂洞察
2. 概念优先，法条佐证——先讲清"为什么"，再引法条作为权威支撑
3. 以考试为导向——每段都应能帮助读者答题

### 图片说明（最重要！）
采用"概念拓展模式"，三步结构：
1. 先抛出"为什么"的问题
2. 借图片讲清制度逻辑和深层原理
3. 延伸到实务或考试要点

禁止使用描述模式，如"PPT 展示了……""PPT 明确了……"。
正确示例：
> 为什么信用证需要 12 步之多？核心在于买卖双方互不信任……
> ![信用证 12 步流程](slides/xxx.png)
> 其中最关键的环节是……

### 法条引用
优先用当前可用的法律数据库或 MCP 查询法条原文；若无专用工具，只引用已可靠确认的条文或要点，不补写不确定原文。引用格式：
> **《XX法》第 N 条**：（法条原文摘要）

引用后必须附要点提炼。条号如有不确定，优先先检索核对；无法核对时标记 `需回看确认`。

### 案例分析
四段结构：事实 → 争点 → 裁判 → 启示

### 答题模板
每个章节末尾附答题模板，格式：
1. 第一步做什么
2. 第二步做什么
3. ……

## 输出要求
- 产出 `{NN}-course_summary.md`（完整学习笔记）和 `{NN}-exam_focus.md`（考试速查手册）
- course_summary 覆盖所有知识点，exam_focus 只保留高频考点

## 工作流程
1. 读取当前 session_notes 了解已有内容
2. 读取对应转写稿获取课堂讨论细节
3. 读取复习资料（如有）获取幻灯片和知识框架
4. 用 Edit 工具逐份补充/改写
5. 完成后报告每份文件的修改摘要和法条引用清单
```

### 3.2 图片验证代理增强 prompt

```
你的任务是验证 {课程名} session_notes 中的图片引用是否准确。

## 验证标准
不仅检查"图片是否存在"，还要检查"图片说明是否符合概念拓展模式"：
- 说明是否以"为什么"的问题开头？
- 是否借图片讲解了制度逻辑？
- 是否延伸到了实务或考试要点？

如果说明只是"PPT 展示了……"的描述模式，需要改写为概念拓展模式。

## 工作流程
1. 用 Grep 搜索所有 `!\[` 图片引用
2. 对每个引用：
   a. 用 Read 打开图片文件，确认存在
   b. 查看图片实际内容
   c. 对比 markdown 中的说明文字
   d. 判断：说明是否采用概念拓展模式？内容是否准确？
3. 如不匹配：在原始幻灯片目录中查找正确图片并修正
4. 如说明为描述模式：改写为概念拓展模式

## 产出
写 `{课程名}_img_verify.md`：
- 验证通过的图片（列表 + 说明质量评级）
- 修正过的图片（原引用 → 修正后 + 修正原因）
- 改写过的说明（原文 → 改写后）
- 缺失的图片（标记位置 + 建议）
```

### 3.3 快速精修代理 prompt（无幻灯片时）

当课程没有 PPT/PDF 复习资料、仅有转写稿时使用：

```
你的任务是对 {课程名} session_notes 第 {起始}-{结束} 讲进行文本深度精修。
本批次没有幻灯片图片，专注于文本内容的深度加工。

## 文件路径
- 当前 session_notes：`{notes_path}`
- 转写稿：`{sessions_path}`

## 精修重点（无图片版）
1. 从转写稿的口语中提取知识点骨架，用书面语重新组织
2. 补充法条引用（优先用可用法律工具核对条号；无专用工具时仅写已可靠确认内容）
3. 从转写稿中提取教师讨论的案例，构建四段分析框架
4. 为每个章节生成答题模板
5. 区分 course_summary（完整笔记）和 exam_focus（考试速查）的内容侧重

## 输出要求
- `{NN}-course_summary.md`：完整学习笔记
- `{NN}-exam_focus.md`：高频考点 + 法条速查表 + 答题模板

## 工作流程
1. 读取转写稿，标记知识点、案例、教师强调
2. 读取当前笔记，了解已有结构
3. 用 Edit 工具补充改写
4. 完成后报告修改摘要
```

## 4. 已知陷阱与经验

- **转写稿中的数字错误**：Whisper 经常把法条号转错（如"第十七条"变成"第七十条"或误成其他数字），有专用法律工具时必须核验；无专用工具时不要猜测补写
- **教师口语中的跳跃**：教师讲课时会突然跳到另一个话题再跳回来，精修时要重新组织逻辑顺序，不要照搬口语的叙述顺序
- **图片引用准确性**：内容补充完成后必须逐张抽查图片引用，曾发现 slide-25 被错误用于展示 slide-30 的内容
- **法条版本问题**：部分法律已经修订（如《公司法》2023 年大修），引用时注意版本；有专用法律工具时以最新版本为准，无专用工具时保守标记 `需回看确认`
- **course_summary 不等于 transcript 的整理版**：精修是"重新写一份笔记"，不是"把口语翻译成书面语"。要加入教师没有明确说但学生应该知道的知识（构成要件、法条体系、学说争议）
